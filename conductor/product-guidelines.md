# Voice Stall - Product Guidelines

## Prose & Documentation Style
- **Technical & Concise**: All internal logs, performance metrics, and application feedback should prioritize speed and clarity over excessive description. Avoid unnecessary "fluff" and keep messages direct.
- **Example-Driven & Practical Docs**: Documentation and README files should be grounded in practical use cases. Use clear commands, code snippets, and step-by-step examples for common tasks (like adding a new profile or troubleshooting CUDA).

## Branding & Visual Tone
- **High-Performance & Modern**: The application should feel like a precision tool for power users. This aligns with the "Deep Night Minimalist" aesthetic: dark, high-contrast, and focused on functional elegance.
- **Privacy Centric**: All branding should reinforce the "Local-First" nature of the application, emphasizing that data remains private and secure.

## User Experience (UX) Principles
- **Silent & Unobtrusive Interaction**: Hotkey actions (start/stop dictation) should not disrupt the user's workflow. Any feedback must be minimal and fast.
- **Subtle Visual Cues**: Provide small, non-intrusive visual indicators (e.g., a change in the tray icon or a tiny floating status bar) to confirm the application's state without being distracting.
- **Efficiency over Frills**: Prioritize the speed of text delivery and the responsiveness of the hotkey system above any non-essential UI animations.
